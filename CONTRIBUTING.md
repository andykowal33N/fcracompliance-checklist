# Contributing

We welcome ideas, feedback, and pull requests that make these projects stronger and more useful.

## How to Contribute
1. **Open an Issue** — Describe the improvement or bug fix you’re proposing.  
2. **Fork the Repo** — Create a feature branch: `feature/<short-name>`.  
3. **Make Changes** — Keep edits focused and well-documented.  
4. **Submit a Pull Request** — Reference the Issue number and summarize your change.

## Development Guidelines
- Use **Python 3.10+**  
- Work in a clean virtual environment  
- Follow consistent formatting and linting (PEP8 recommended)  
- Include or update docstrings where applicable  
- Keep commits small and messages clear  

## Communication
Be professional, concise, and constructive in all discussions.  
We’re here to build reliable, transparent, and practical tools for data analytics and risk intelligence.
